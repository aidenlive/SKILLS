Common ARIA Patterns Reference

Copy-paste ready examples of common ARIA patterns for accessible web components.

1. Disclosure (Accordion/Collapsible)

Pattern

A widget that allows users to show/hide content.

<button
  aria-expanded="false"
  aria-controls="panel-1"
>
  Show More
</button>

<div id="panel-1" hidden>
  Hidden content
</div>

2. Tabs

Pattern

A set of layered sections of content, only one visible at a time.

<div role="tablist" aria-label="Content sections">
  <button
    role="tab"
    aria-selected="true"
    aria-controls="panel-1"
    id="tab-1"
  >
    Tab 1
  </button>
  <button
    role="tab"
    aria-selected="false"
    aria-controls="panel-2"
    id="tab-2"
  >
    Tab 2
  </button>
</div>

<div role="tabpanel" id="panel-1" aria-labelledby="tab-1">
  Panel 1 content
</div>

<div role="tabpanel" id="panel-2" aria-labelledby="tab-2" hidden>
  Panel 2 content
</div>

3. Alert / Live Region

Pattern

Dynamically updating content that should be announced to screen readers.

<!-- Error alert (assertive) -->
<div role="alert" aria-live="assertive">
  Error: Please correct the form fields.
</div>

<!-- Status message (polite) -->
<div role="status" aria-live="polite">
  3 items added to cart
</div>

<!-- Loading state -->
<div aria-live="polite" aria-busy="true">
  Loading...
</div>

4. Dialog / Modal

Pattern

A modal dialog that requires user interaction.

<div
  role="dialog"
  aria-modal="true"
  aria-labelledby="dialog-title"
  aria-describedby="dialog-desc"
>
  <h2 id="dialog-title">Confirm Action</h2>
  <p id="dialog-desc">
    Are you sure you want to proceed?
  </p>
  <button>Cancel</button>
  <button>Confirm</button>
</div>

Key requirements:

  • Focus trap (Tab cycles within modal)
  • Escape key closes modal
  • Focus returns to trigger element on close
  • Body scroll prevented when open

5. Landmark Roles

Pattern

Semantic regions that help screen reader users navigate.

<!-- Use semantic HTML when possible -->
<header>...</header>  <!-- role="banner" implicit -->
<nav>...</nav>        <!-- role="navigation" implicit -->
<main>...</main>      <!-- role="main" implicit -->
<aside>...</aside>    <!-- role="complementary" implicit -->
<footer>...</footer>  <!-- role="contentinfo" implicit -->

<!-- Use ARIA roles when semantic HTML isn't available -->
<div role="search">
  <input type="search" aria-label="Search site">
</div>

<!-- Multiple landmarks of same type need labels -->
<nav aria-label="Main navigation">...</nav>
<nav aria-label="Footer navigation">...</nav>

6. Form Labels and Descriptions

Pattern

Properly labeled and described form fields.

<!-- Explicit label -->
<label for="email">Email</label>
<input
  type="email"
  id="email"
  aria-describedby="email-help"
  required
  aria-required="true"
>
<span id="email-help">
  We'll never share your email
</span>

<!-- Error state -->
<label for="password">Password</label>
<input
  type="password"
  id="password"
  aria-describedby="password-error"
  aria-invalid="true"
>
<span id="password-error" role="alert">
  Password must be at least 8 characters
</span>

7. Button States

Pattern

Buttons with clear states and purposes.

<!-- Toggle button -->
<button aria-pressed="false">
  Notifications
</button>

<!-- Icon-only button -->
<button aria-label="Close dialog">
  <svg aria-hidden="true">...</svg>
</button>

<!-- Disabled button -->
<button disabled>
  Submit
</button>
